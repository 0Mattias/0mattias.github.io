bettermemory

v3.14 · updated July 3, 2026 · MIT · works with Claude Code, Cursor, and any MCP client

Memory tools for AI agents are good at storing facts and silent about whether a fact is still true. Files move, projects pivot, preferences change, and the stored note from March reads as gospel in November.

bettermemory is built around that gap. It is a local MCP server: plain markdown on disk, 25 tools, no cloud, no database. Every memory it serves comes back with a verdict on whether the fact still holds, so the agent checks before it trusts.

the verdict, in four steps

{
  "snippet": "Auth middleware lives in src/auth/middleware.py",
  "relevance": "high",
  "staleness_verdict": "spot_check_required",
  "path_drift": { "missing": ["src/auth/middleware.py"] },
  "commit_drift_count": 31
}

This is the real shape of a search hit. The agent branches on the verdict before it leans on the snippet.

sixty seconds, offline

The walkthrough above is scripted. bettermemory try runs the real machinery in a throwaway store: it writes a memory citing a file, deletes the file, and the next search comes back flagged stale.

uv tool install bettermemory
bettermemory try

what it changes

  • It checks before it trusts. Every hit carries a staleness verdict: calendar age, moved file paths, and commits landed since the fact was last confirmed.
  • It asks before it remembers you. Retrieval is opt-in per turn, claims about you stage for your sign-off, and pasted secrets never reach disk.
  • Every use leaves a record. The server logs which sentence a memory shaped, and flags the searches the model should have run but skipped.
  • It carries state across agents. Loops and subagent swarms journal into one shared store; the next session picks up mid-thought instead of cold.

recent releases

v3.14

July 3, 2026

The measurement release. Three telemetry layers were quietly mis-measuring: long natural-language queries got labeled "medium" on strong matches, busy turns marked their own retrievals as used before the reply even existed, and one long turn could log the same missed retrieval seven times. Every audited turn now records replayable ranking features, retrievals settle when the reply actually exists, and repeats dedup — the effectiveness numbers measure the system, not the instrumentation.

v3.13

July 3, 2026

The recall-repair release. Four tokenizer fixes had quietly respelled the on-disk search index, so a query like "todo" or "cookie" could miss the exact memories ranked highest; the index now heals itself on first touch, and a tokenizer fingerprint makes that class of silent drift impossible to reintroduce. Sixteen developer words rescued from the stopword lists, and one-character CJK queries finally hit.

v3.12

July 2, 2026

Tokenizer v2. Memories written in Japanese, Korean, or Chinese used to store fine and then stay unfindable; character bigrams fix search, dedup, and the groundedness gate in one pass. Plural folding lands too ("standups" now finds "standup"), plus Swedish, German, French, and Spanish stopword lists.

Full history in the changelog.
