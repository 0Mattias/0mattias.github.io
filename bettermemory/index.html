bettermemory

v3.10 · updated June 11, 2026 · MIT · works with Claude Code, Cursor, and any MCP client

Storing facts is the easy half of agent memory; knowing a fact has held up weeks and forty commits later is the hard half. bettermemory is built around the hard half: plain markdown and a local MCP server underneath, an extension of the model in behavior. A markdown folder and 25 MCP tools that give the model a sense of time.

It changes four things about how the model behaves:

  • It checks before it trusts. Every hit carries a staleness verdict: calendar age, moved file paths, and commits landed since the fact was last confirmed.
  • It asks before it remembers you. Retrieval is opt-in per turn, claims about you stage for your sign-off, and pasted secrets never reach disk.
  • Every use leaves a record. The server logs which sentence a memory shaped, and flags the searches the model should have run but skipped.
  • It carries state across agents. Loops and subagent swarms journal into one shared store; the next session picks up mid-thought instead of cold.

Here is what one search hit looks like to the model. The verdict is the whole point:

{
  "snippet": "Auth middleware lives in src/auth/middleware.py",
  "relevance": "high",
  "staleness_verdict": "spot_check_required",
  "path_drift": { "missing": ["src/auth/middleware.py"] },
  "commit_drift_count": 12
}

The verdict tracks what happened since the fact was stored. Move the cited file and it stops trusting it; verify it and it goes back to fresh.

Install

Claude Code:

/plugin marketplace add 0Mattias/bettermemory
/plugin install bettermemory@bettermemory

Elsewhere: uv tool install bettermemory then bettermemory init --client <name>
