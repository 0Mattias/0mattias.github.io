bettermemory

v3.27 · updated July 22, 2026 · MIT · works with Claude Code, Cursor, and any MCP client

Memory tools for AI agents are good at storing facts and silent about whether a fact is still true. Files move, projects pivot, preferences change, and the stored note from March reads as gospel in November. A confidently remembered stale fact is worse than no memory at all.

bettermemory is built around that gap. It is a local MCP server (plain markdown on disk, 25 tools, no cloud, no database), but the store is the substrate, not the point. The point is trust: a stored fact is treated as a claim that has to keep earning belief. It is verified against the filesystem and git before the agent relies on it, attributed down to the sentence it shaped when it gets used, measured for whether it actually helped, and owed a public postmortem when a verdict is wrong.

the verdict, in four steps

{
  "snippet": "Auth middleware lives in src/auth/middleware.py",
  "relevance": "high",
  "staleness_verdict": "spot_check_required",
  "path_drift": { "missing": ["src/auth/middleware.py"] },
  "commit_drift_count": 31
}

This is the real shape of a search hit. The agent branches on the verdict before it leans on the snippet.

sixty seconds, offline

The walkthrough above is scripted. bettermemory try runs the real machinery in a throwaway store: it writes a memory citing a file, deletes the file, and the next search comes back flagged stale.

uv tool install bettermemory
bettermemory try

what it changes

  • It checks before it trusts. Every hit carries a staleness verdict: calendar age, moved file paths, and commits landed since the fact was last confirmed.
  • It asks before it remembers you. Retrieval is opt-in per turn, claims about you stage for your sign-off, and pasted secrets never reach disk.
  • Every use leaves a record. The server logs which sentence a memory shaped, and flags the searches the model should have run but skipped.
  • It measures whether it's helping. A built-in eval reports how often retrieved memory shaped a reply and how often it should have been consulted but wasn't. Numbers, not vibes, published with the repo, unflattering ones included.
  • It answers for wrong verdicts. A memory that misleads an agent is owed a public postmortem; the incidents ledger ships in the repo, and it is meant to be embarrassing to add to.
  • It carries state across agents. Loops and subagent swarms journal into one shared store; the next session picks up mid-thought instead of cold.

recent releases

v3.27

July 20, 2026

The disclosure release. Store roots were created with the caller's umask, and a memory's filename embeds the first ~43 characters of its summary, so a world-listable root handed any local account the gist of the whole store from ls alone. Roots are now created private and healed on open (POSIX, best-effort; doctor reports anything a filesystem refuses). Also fixed: BM25 priced term rarity against the wrong collection. Document frequencies now come from the index, over exactly the scoped set the search ranks, because whole-store numbers describe memories the caller cannot retrieve.

v3.26

July 20, 2026

The guarantees release. Event-log rotation could silently drop events under concurrent writers; archives are now shard-partitioned with a real ordering guarantee, and stores upgraded mid-line keep a best-effort fallback. sync gained guards on every path that could commit conflict markers as resolved content (a class found live: a push had shipped markers to the remote with no error raised). And shipped prose became testable: a checker extracts path, symbol, and count claims from the docs and asserts each against the repo, with a ratcheted allowlist so an exemption cannot outlive the defect it excuses.

v3.25

July 19, 2026

The provenance release. migrate origin --repair fixes memories whose origin was captured wrong: a write made outside a checkout claimed no repo at all, so auto-scoping quietly hid it from the project it belonged to (--keep-global preserves the ones that really are global). Two patches rode the same day: a Windows-only durability gap in the atomic write path, and a path-drift false positive that made healthy web-app memories look stale because bare application routes read as missing files.

Full history in the changelog.
